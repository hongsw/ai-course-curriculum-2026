# 2027 개선안 / Lessons & Improvements

> 2026 운영 데이터(제출률·점수 분포·재평가 패턴)에서 도출. 우선순위 P0(필수)~P2(권장).

## P0 — 제출 규격 표준화
**근거**: 명확한 산출물 규격(repo+보고서+영상)이 있던 과목은 제출 사고(0점) 0건, 없던 과목은 편차가 컸다.
**개선**: 모든 프로젝트 제출을 단일 템플릿(`제출.md`: repo URL·보고서·영상 링크 고정 필드)으로. 댓글·파일 혼용 금지. 제출 직후 자동 체크(repo 공개·README 필수 섹션·영상 링크 유효성).

## P0 — 주차 주제·평가 기준 사전 명문화
**근거**: 일부 주차가 게시판 본문 없이 "과제"로만 게시돼 주제를 제출물로 사후 확정해야 했다.
**개선**: 각 주차 페이지에 학습목표 1줄 + 평가 루브릭 3~5항을 사전 게시.

## P1 — 대형 기말을 마일스톤 3개로 분산
**근거**: 중반 이후 난이도 상승 구간에서 제출률 하락.
**개선**: 기말을 주제 선정 → 중간 데모 → 최종으로 분해. 공통 **최소 재현 스타터 repo**(로그·export 자리 포함) 제공 → 최소 규모 제출 예방.

## P1 — 실질 5기준 채점 루브릭
**근거**: 표면 분량·추출 품질(비원어민·PDF 합자 깨짐)로 인한 과소평가를 재평가로 다수 정정.
**개선**: ①직접 수행 증거(실측·스크린샷·로그) ②결과 vs 계획 서술 ③예시 재사용 여부 ④발표/기록 현장성 ⑤문헌 요약 대비 자기 분석 — 이번 학기 재평가에서 검증된 기준을 명시 루브릭화.

## P2 — 두 과목 연계 캡스톤
**개선**: 파인튜닝한 모델(NLP)을 에이전트 서비스(OSS)로 배포하는 수직 통합 트랙 — 한 학기 산출물이 다음 과목의 입력이 된다.

## P2 — Git 협업의 실질 평가
**근거**: 2인 1조 요건에서 실제 협업 팀은 소수, 나머지는 개인+셀프 PR.
**개선**: 협업을 강제(팀 배정)하거나 개인 트랙을 명시 허용해 중간지대를 제거.

## 한 문장 요약
**규격을 표준화하고(P0), 큰 과제를 쪼개고(P1), 분량이 아니라 실질로 채점하라(P1)** — 이 셋이 제출 사고와 평가 편차를 가장 크게 줄인다.
