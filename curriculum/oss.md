# 오픈소스SW응용 — 주차별 커리큘럼 (Open-Source SW Applications)

**과목 성격**: "AI 코딩 도구로 오픈소스를 다루는 법"을 손으로 익히는 실습 중심 과목. Git·터미널에서 출발해 컨테이너·멀티에이전트·MCP·로컬 LLM 스택을 거쳐 **Pi 기반 AI 에이전트 웹서비스**로 마무리하는 상향식(bottom-up) 아크.

## 주차별 설계

| 주차 | 학습 목표 | 산출물 |
|---|---|---|
| 1–2 | Git 이력 관리 + AI 코딩 도구 첫 사용 | Git 기록 + 보고서 |
| 2 | 터미널 기초 / 스크립트 작성 | 터미널 실습 · 카운트 스크립트 |
| 3 | opencode + **2인 1조 협업** | 팀 GitHub repo |
| 4 | AI 코딩 도구 실습 | 실습 산출물 |
| 5 | 컨테이너 기초 | Dockerfile → 이미지 → 빌드 → 실행 |
| 6 | **멀티에이전트 팀 설계** | 서브에이전트 3개+ 다단계 추론 팀 |
| 7 | 오픈소스로 문제해결 서비스 (중간) | GitHub repo + 발표 슬라이드 |
| 10 | **자작 MCP 서버** | MCP 함수 → opencode 연결 → 호출 로그 |
| 11 | AI 주도 개발(계획→구현→검증) | 코드·실행결과·아키텍처·테스트 |
| 12 | **오픈 로컬 LLM 스택** | 레이어 정의 + 오픈소스 매핑 + 실행환경 구축 |
| 15–16 | **Pi 기반 AI Agent 웹서비스** (기말) | Skill + MCP + Extension + Web UI |

## 기말 평가 루브릭 (100점)

| 항목 | 배점 |
|---|---|
| 아이디어·문제정의 | 20 |
| Pi(에이전트 런타임) 활용 | 20 |
| Skill · MCP · Extension 구현 | 25 |
| Web UI (CLI 불인정) | 15 |
| repo 품질 | 10 |
| 보고서·발표 영상 | 10 |

## 설계 관찰

- 도구 온보딩(Git→터미널→opencode)에서 **실제 산출물 중심**(Docker 이미지 → MCP 서버 → 에이전트 팀 → 배포형 서비스)으로 난이도가 단계적으로 상승.
- 기말이 "AI API 호출"이 아니라 **Skill·MCP·Extension을 결합한 서비스형 프로토타입**을 요구 — 2026년 에이전트 개발 실무와 직결.
- 중반 이후(자작 MCP·AI 주도 개발·로컬 LLM) 난이도 상승 + 개인 과제 전환 구간에서 제출률이 하락 → 2027 개선안의 마일스톤 분산으로 대응.
